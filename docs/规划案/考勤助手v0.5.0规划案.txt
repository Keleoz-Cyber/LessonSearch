考勤记录 App v0.5.0 功能规划案
一、版本目标
本次版本主要实现以下内容：
1.	记名名单提交功能 
2.	周名单汇总功能 
3.	用户角色分级制度 
本版本的核心目标是建立完整的周考勤处理流程，即：
成员提交记名名单 → 管理员审核 → 系统汇总 → 导出周考勤表
同时，为后续版本扩展历史归档、统计分析、提醒机制等功能打下基础。
________________________________________
二、角色与权限设计
系统设定三类账户角色：
•	超级管理员 
•	管理员 
•	成员（普通成员） 
权限等级如下：
超级管理员 > 管理员 > 成员
1. 成员权限
成员可使用以下功能：
•	使用基础查课与记名功能 
•	提交记名名单 
•	查看本人提交记录及审核状态 
•	撤销本人“待审核”状态的提交记录 
•	查看周名单汇总结果（只读） 
2. 管理员权限
管理员可使用以下功能：
•	拥有成员全部权限 
•	查看所有成员提交情况 
•	审核成员提交记录 
•	查看周汇总名单 
•	对本周通过记录进行汇总 
•	导出 Excel 周考勤表 
3. 超级管理员权限
超级管理员可使用以下功能：
•	拥有管理员全部权限 
•	修改账户角色 
•	管理账户信息 
•	管理账户记录及系统内相关数据 
________________________________________
三、周次规则
1. 周次定义方式
系统中的“本周”不采用自然周自动计算，而是采用 手动设定起始周 的方式进行统一计算。
例如：
•	设定 3 月 2 日为第 1 周 
•	后续系统按周期自动推算： 
o	第 2 周 
o	第 3 周 
o	… 
o	第 X 周 
2. 周次用途
所有以下业务都必须基于该周次规则进行处理：
•	记名名单提交 
•	提交审核 
•	周名单汇总 
•	Excel 周表导出 
•	历史周数据保留 
3. 设计要求
系统应支持管理员或超级管理员预先设定“第 1 周起始日期”，后续周次由系统自动递增计算。
________________________________________
四、功能一：记名名单提交
1. 功能定位
“记名名单提交”功能用于让成员将本周已完成的记名查课记录上传到服务端，进入审核流程，为后续周名单汇总提供数据来源。
2. 权限范围
仅 成员及以上角色 可使用。
3. 提交流程
成员点击“记名名单提交”按钮后，进入提交界面，系统展示当前可提交的记名记录列表。
用户可进行以下操作：
•	多选需要提交的记名记录 
•	查看选中记录的详细信息 
•	进行二次确认 
•	确认后提交到服务端 
4. 可提交记录范围
仅允许提交满足以下条件的记名记录：
•	记录类型为记名记录 
•	状态不是“已放弃” 
•	状态不是“进行中” 
•	记录属于当前设定周次范围内 
5. 重复提交限制
系统 不允许重复提交完全相同的记录。
若用户提交的记录与已提交记录完全一致，系统需进行拦截并提示，例如：
“存在重复提交记录，无法重复提交相同内容。”
这里的“完全相同记录”建议至少基于以下信息判定：
•	记录所属周次 
•	班级信息 
•	记录对象 
•	提交内容 
•	记录标识 
6. 提交后状态
记录提交后，服务端需独立存储，并赋予审核状态：
•	待审核 
•	审核通过 
•	审核未通过 
以上状态对提交用户可见。
7. 成员可见范围
成员仅可查看自己的提交记录及其状态，包括：
•	提交时间 
•	提交内容摘要 
•	当前审核状态 
•	是否允许撤销 
8. 撤销规则
成员仅可撤销 本人提交且状态为“待审核” 的记录。
撤销后：
•	该记录从审核队列中移除 
•	不再参与本周汇总 
以下情况不可撤销：
•	已审核通过 
•	已审核未通过 
9. 审核通过后的限制
一旦记录被审核通过，则该记录：
•	不可撤销 
•	不可修改 
该规则为强约束，用于保证周汇总数据稳定性与导出一致性。
10. 功能说明
该功能的本质是为“周名单汇总”提供审核前数据来源，并非直接生成最终考勤结果。
________________________________________
五、功能二：周名单汇总
1. 功能定位
周名单汇总功能用于对本周所有 审核通过 的记名提交进行统一整合，生成本周正式考勤汇总结果。
2. 分角色展示
成员视图
成员仅可查看周名单汇总结果，权限为：
•	只读查看 
•	不可审核 
•	不可修改 
•	不可导出 
管理员 / 超级管理员视图
管理员与超级管理员可见以下三个功能入口：
1.	周汇总名单查看 
2.	审核记录 
3.	记录汇总导出 
________________________________________
六、功能三：审核记录
1. 功能定位
管理员对成员提交的记名名单进行审核，决定其是否进入本周汇总。
2. 审核对象
管理员可查看所有成员在当前周次内提交的记录，包括：
•	提交人 
•	提交时间 
•	提交内容详情 
•	当前审核状态 
3. 审核操作
管理员可执行以下操作：
•	通过 
•	不通过 
4. 审核结果规则
•	审核通过：该记录进入本周有效汇总数据 
•	审核未通过：该记录不进入本周汇总 
5. 审核限制
审核范围仅限当前周次内的记录，不同周次数据应分开处理，不得混淆。
________________________________________
七、功能四：记录汇总与导出
1. 汇总逻辑
管理员可将本周全部 审核通过 的名单进行一键整合，生成周考勤预览表。
2. 汇总预览
系统应先生成预览表，供管理员确认；确认后方可正式导出。
3. 导出格式
导出文件为 Excel 周考勤表。
建议命名格式：
第 X 周周考勤表.xlsx
4. 导出字段
导出的周考勤表至少包含：
•	姓名 
•	学号 
•	班级 
•	迟到 / 早退次数（统一计入“迟到”） 
•	旷课次数（统一计入“缺勤”） 
•	累计 
5. 累计规则
累计规则如下：
•	2 次迟到 = 1 次累计 
•	1 次旷课 = 1 次累计 
系统应自动按统一规则计算累计值。
6. 排序规则
导出名单需按以下顺序排列：
1.	专业 
2.	班级 
3.	名单顺序 
确保周表结构清晰、统一、便于管理与查阅。
7. 导出保留规则
所有导出的周考勤表需 保留历史记录，不得因新一周开始而覆盖旧周数据。
也就是说：
•	第 1 周导出后保留 
•	第 2 周导出后单独保留 
•	… 
•	第 X 周导出后继续累计保留 
便于后续查询、复核与归档管理。
________________________________________
八、成员提交情况统计
管理员应可查看本周各成员提交情况，用于督促与管理。
建议展示内容包括：
•	成员姓名 / 账号 
•	是否已提交 
•	提交次数 
•	最近提交时间 
•	当前审核状态概览 
特殊要求
对 未提交成员 应进行重点标注，便于管理员快速识别并跟进。
________________________________________
九、数据周期规则
系统中的提交、审核、汇总、导出，均需严格按照“周次”进行隔离。
具体要求如下：
•	每一周形成独立的数据集合 
•	每周生成独立周考勤表 
•	新一周开始后，进入新的考勤周期 
•	历史周数据保留，不覆盖、不混淆 
•	所有操作需明确归属到“第 X 周”
