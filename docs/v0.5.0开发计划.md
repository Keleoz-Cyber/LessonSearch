## 十二、v0.5.0 功能开发计划

### 12.1 版本目标

本次版本主要实现以下内容：
1. 记名名单提交功能
2. 周名单汇总功能
3. 用户角色分级制度

本版本的核心目标是建立完整的周考勤处理流程，即：
**成员提交记名名单 → 管理员审核 → 系统汇总 → 导出周考勤表**

**重要说明：**
- ✅ 记名（name_check）任务需要提交审核
- ❌ 点名（roll_call）任务不需要提交
- 记名任务包含多个班级的完整考勤情况，适合提交汇总
- 点名任务是单人点名，主要用于快速点名场景，不参与周汇总

### 12.2 角色系统实现

**角色分类：**
- 超级管理员（super_admin）
- 管理员（admin）
- 成员（member）

**实现细节：**
- 超级管理员账号：使用已注册账号设置 role='super_admin'（不是创建新账号）
- 邀请码功能：仅用于注册认证，不区分角色
- 角色分配：超级管理员可分配角色
- User 表修改：增加 role 字段

**权限控制：**
- API 层：中间件/依赖注入检查角色权限
- 前端层：路由守卫 + 页面按钮级别控制

**详细权限矩阵：**

| 功能 | member | admin | super_admin | 说明 |
|------|--------|-------|-------------|------|
| 查看自己的任务 | ✅ | ✅ | ✅ | 所有角色都有 |
| 创建任务 | ✅ | ✅ | ✅ | 所有角色都有 |
| 查看自己的记录 | ✅ | ✅ | ✅ | 所有角色都有 |
| 提交考勤记录 | ✅ | ✅ | ✅ | 所有角色都有 |
| 撤销自己的提交 | ✅ | ✅ | ✅ | 仅自己未审核的提交 |
| 查看所有人的任务 | ❌ | ✅ | ✅ | 管理员及以上 |
| 查看所有人的记录 | ❌ | ✅ | ✅ | 管理员及以上 |
| 查看所有用户信息 | ❌ | ✅ | ✅ | 管理员及以上 |
| 查看统计数据 | ❌ | ✅ | ✅ | 管理员及以上 |
| 审核提交记录 | ❌ | ✅ | ✅ | 管理员及以上 |
| 导出周考勤汇总 | ❌ | ✅ | ✅ | 管理员及以上 |
| 分配用户角色 | ❌ | ❌ | ✅ | 仅超级管理员 |
| 设置周次起始日期 | ❌ | ❌ | ✅ | 仅超级管理员 |
| 重置学期周次 | ❌ | ❌ | ✅ | 仅超级管理员 |
| 管理邀请码 | ❌ | ❌ | ✅ | 仅超级管理员 |

**超级管理员特殊权限说明：**

超级管理员拥有最高权限，可以查看和管理所有用户的所有数据：

1. **查看权限：**
   - 查看所有用户的任务列表
   - 查看所有考勤记录
   - 查看所有提交记录
   - 查看所有用户信息（id, email, nickname, role, created_at, last_login_at）
   - 查看统计数据（如：每个用户查课次数、任务完成情况）

2. **管理权限：**
   - 分配/修改用户角色（提升为 admin 或降级为 member）
   - 设置学期第1周起始日期
   - 重置学期周次配置
   - 创建/管理邀请码
   - 审核所有提交记录
   - 导出任意周次的考勤汇总表

3. **API 示例：**
```python
# 超级管理员查看所有任务
@router.get("/admin/tasks")
def list_all_tasks(super_admin: User = Depends(get_current_super_admin)):
    return db.query(AttendanceTask).all()

# 超级管理员查看所有记录
@router.get("/admin/records")
def list_all_records(super_admin: User = Depends(get_current_super_admin)):
    return db.query(AttendanceRecord).all()

# 超级管理员查看用户列表
@router.get("/admin/users")
def list_all_users(super_admin: User = Depends(get_current_super_admin)):
    return db.query(User).all()

# 超级管理员统计数据
@router.get("/admin/stats")
def get_statistics(super_admin: User = Depends(get_current_super_admin)):
    return {
        "total_users": db.query(User).count(),
        "total_tasks": db.query(AttendanceTask).count(),
        "tasks_by_user": db.query(
            User.email,
            func.count(AttendanceTask.id)
        ).join(AttendanceTask).group_by(User.id).all()
    }
```

### 12.3 周次配置实现

**规则：**
- 权限：仅超级管理员可设定第1周起始日期
- 学期结束处理：手动重置（创建新学期配置）
- 周次计算：根据起始日期动态计算当前周次

**新学期设置流程：**

超级管理员在服务器手动操作：

```sql
-- 1. 禁用旧学期配置
UPDATE week_config SET is_active = FALSE;

-- 2. 创建新学期配置
INSERT INTO week_config (
    start_date,
    semester_name,
    is_active
) VALUES (
    '2026-09-01',           -- 新学期第1周起始日期
    '2026-2027学年第一学期', -- 学期名称
    TRUE                    -- 设置为当前活跃学期
);
```

操作时机：
- 学期开始前（开学第一周之前）
- 系统会自动使用最新的 is_active=TRUE 配置计算周次

```sql
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**说明：**
- is_active: 标识当前活跃学期配置（只有一个为 TRUE）
- 新学期设置：创建新记录并设置 is_active=TRUE，旧记录自动设为 FALSE
- 周次计算：使用当前 is_active=TRUE 的配置计算
- 历史记录：保留历史学期配置用于查询历史数据

### 12.6 Excel导出模板格式

**文件名：** 第X周周考勤表.xlsx

**列结构（7列）：**

| 列号 | 列名 | 数据类型 | 说明 |
|------|------|----------|------|
| 1 | 序号 | 整数 | 自动编号（从1开始） |
| 2 | 姓名 | 字符串 | 学生姓名 |
| 3 | 班级 | 字符串 | 班级名称（如：电信2402） |
| 4 | 学号 | 字符串 | 完整学号（如：241040200219） |
| 5 | 迟到/早退 | 整数 | 迟到或早退次数 |
| 6 | 旷课 | 整数 | 旷课次数 |
| 7 | 累计 | 整数 | 累计次数（公式计算） |

**字段映射关系：**

| Excel列名 | APP记录状态 | 说明 |
|-----------|-------------|------|
| 迟到/早退 | late | APP中标记为"迟到"状态的记录数 |
| 旷课 | absent | APP中标记为"缺勤"状态的记录数 |
| 请假 | - | 不计入Excel导出 |
| 其他 | - | 不计入Excel导出 |
| 已到 | - | 不计入Excel导出 |

**累计公式：**
```
=ROUNDDOWN(迟到/早退/2 + 旷课, 0)
```
即：
- 2次迟到 = 1次累计
- 1次旷课 = 1次累计

**数据排序规则：**

1. **专业排序**（按 major.short_name 字母顺序）：
   - 空信（Kongxin）
   - 通信（Tongxin）
   - 物联网（Wulianwang）
   - 电信（XinDian）
   - 计科（JiKe）
   - 实际排序：使用 ORDER BY major.short_name ASC

2. **班级排序**（按 class.class_code 数值升序）：
   - 电信2401 → 电信2402 → 电信2403
   - 实际排序：使用 ORDER BY CAST(class.class_code AS SIGNED) ASC

3. **学号排序**（按 student.student_no 升序）：
   - 实际排序：使用 ORDER BY student.student_no ASC

**完整 SQL 排序示例：**
```sql
SELECT 
    s.name, 
    c.display_name as class_name, 
    s.student_no,
    ...
FROM students s
JOIN classes c ON s.class_id = c.id
JOIN majors m ON c.major_id = m.id
ORDER BY 
    m.short_name ASC,          -- 专业排序
    CAST(c.class_code AS SIGNED) ASC,  -- 班级排序
    s.student_no ASC           -- 学号排序
```

**模板示例：**
```
序号 | 姓名   | 班级     | 学号         | 迟到/早退 | 旷课 | 累计
1    | 叶知秋 | 电信2402 | 241040200219 | 0         | 1    | 1
2    | 陈冠锦 | 电信2403 | 241040200302 | 0         | 1    | 1
```

- 累计列使用Excel公式（非硬编码数值）

```sql
    UNIQUE KEY uk_week (week_number)
);

-- 3. 设置超级管理员（需要手动修改邮箱）
-- UPDATE users SET role = 'super_admin' WHERE email = '管理员邮箱';
```

**客户端兼容性：**

- ✅ Token 生成逻辑不变（JWT payload 添加 role 字段）
- ✅ API 接口向后兼容（新增字段不影响旧接口）
