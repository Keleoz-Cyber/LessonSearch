1. **拆分 models.py**

   ```bash
   mv models.py app/models/
   ```

2. **拆分 schemas.py**
   ```bash
   mv schemas.py app/schemas/
   ```

### 5.2 回滚步骤

**如果重构失败：**

```bash
# 切回主分支
git checkout main

# 删除开发分支
git branch -D feature/v0.5.0

# 重新创建开发分支
git checkout -b feature/v0.5.0-backup

# 从稳定版本重新开始
```

**部分回滚：**

```bash
# 回滚到某个 commit
git reset --hard <commit-hash>

# 或创建反向 commit
git revert <commit-hash>
```

## 十二、v0.5.0 功能开发计划

### 12.1 版本目标

本次版本主要实现以下内容：
1. 记名名单提交功能
2. 周名单汇总功能
3. 用户角色分级制度

本版本的核心目标是建立完整的周考勤处理流程，即：
**成员提交记名名单 → 管理员审核 → 系统汇总 → 导出周考勤表**

### 12.2 角色系统实现

**角色分类：**
- 超级管理员（super_admin）
- 管理员（admin）
- 成员（member）

**实现细节：**
- 超级管理员初始账号：服务器直接创建
- 邀请码功能：仅用于注册认证，不区分角色
- 角色分配：超级管理员可分配角色
- User 表修改：增加 role 字段

**权限控制：**
- API 层：中间件/依赖注入检查角色权限
- 前端层：路由守卫 + 页面按钮级别控制

### 12.3 周次配置实现

**规则：**
- 权限：仅超级管理员可设定第1周起始日期
- 学期结束处理：手动重置
- 周次计算：根据起始日期动态计算当前周次

**数据库设计：**
```sql
CREATE TABLE week_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    start_date DATE NOT NULL,
    semester_name VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 12.4 数据库设计

**用户表修改：**
```sql
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'member';
```

**提交记录表：**
```sql
CREATE TABLE submissions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    week_number INT NOT NULL,
    status VARCHAR(20) DEFAULT 'pending', -- pending/approved/rejected
    reviewer_id INT,
    review_time DATETIME,
    review_note TEXT,
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (reviewer_id) REFERENCES users(id)
);
```

**提交-考勤记录关联表：**
```sql
CREATE TABLE submission_records (
    id INT PRIMARY KEY AUTO_INCREMENT,
    submission_id INT NOT NULL,
    record_id INT NOT NULL,
    FOREIGN KEY (submission_id) REFERENCES submissions(id),
    FOREIGN KEY (record_id) REFERENCES attendance_records(id)
);
```

### 12.5 功能实现细节

**提交记录：**
- 包含：查课记录功能所有所需字段
- 根据 Excel 导出内容判断所需字段
- 不上传图片/附件

**撤销记录：**
- 撤销待审核记录后：数据保留本地
- 从审核队列移除
- 不参与本周汇总

**审核流程：**
- 审核记录：记录审核人、审核时间
- 审核不通过原因：可选填写

**导出功能：**
- 支持按周次导出历史周考勤表
- 导出字段：姓名、学号、班级、迟到次数、旷课次数、累计
- 累计规则：2次迟到=1累计，1次旷课=1累计

### 12.6 Excel导出模板格式

**文件名：** 第X周周考勤表.xlsx

**列结构（7列）：**

| 列号 | 列名 | 数据类型 | 说明 |
|------|------|----------|------|
| 1 | 序号 | 整数 | 自动编号（从1开始） |
| 2 | 姓名 | 字符串 | 学生姓名 |
| 3 | 班级 | 字符串 | 班级名称（如：电信2402） |
| 4 | 学号 | 字符串 | 完整学号（如：241040200219） |
| 5 | 迟到/早退 | 整数 | 迟到或早退次数 |
| 6 | 旷课 | 整数 | 旷课次数 |
| 7 | 累计 | 整数 | 累计次数（公式计算） |

**字段映射关系：**

| Excel列名 | APP记录状态 | 说明 |
|-----------|-------------|------|
| 迟到/早退 | late | APP中标记为"迟到"状态的记录数 |
| 旷课 | absent | APP中标记为"缺勤"状态的记录数 |
| 请假 | - | 不计入Excel导出 |
| 其他 | - | 不计入Excel导出 |
| 已到 | - | 不计入Excel导出 |

**累计公式：**
```
=ROUNDDOWN(迟到/早退/2 + 旷课, 0)
```
即：
- 2次迟到 = 1次累计
- 1次旷课 = 1次累计

**数据排序规则：**
1. 专业排序（物联网、电信、空信、计科、通信等）
2. 班级排序（按班级编号）
3. 学号排序（按学号升序）

**模板示例：**
```
序号 | 姓名   | 班级     | 学号         | 迟到/早退 | 旷课 | 累计
1    | 叶知秋 | 电信2402 | 241040200219 | 0         | 1    | 1
2    | 陈冠锦 | 电信2403 | 241040200302 | 0         | 1    | 1
```

**技术实现：**
- 使用 `openpyxl` 或 `pandas` 生成Excel
- 累计列使用Excel公式（非硬编码数值）
- 支持下载到客户端本地
