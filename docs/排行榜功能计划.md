# 排行榜功能实现计划

> 版本：v0.5.5 | 更新日期：2026-04-13

---

## 一、功能概述

排行榜模块用于展示各班级在统计周期内的考勤异常情况排名，包含：
- 3种统计周期：近7天、近30天、总榜
- 3种榜单类型：综合异常分数榜、缺勤率榜、缺勤人次榜
- 共9个排行榜视图

---

## 二、核心配置

### 1. 综合异常分数计算公式

```
综合异常分数 = Σ(状态次数 × 权重) / 应到人次
```

### 2. 状态权重配置

| 状态 | 权重 | 说明 |
|------|------|------|
| 缺勤 (absent) | 1.0 | 最严重 |
| 请假 (leave) | 0.5 | 中等 |
| 迟到 (late) | 0.3 | 较轻 |
| 其他 (other) | 0.2 | 最轻 |

### 3. 缺勤率计算公式

```
缺勤率 = 缺勤人次 / 应到人次 × 100%
```

### 4. 缺勤人次定义

当前统计周期内，该班级所有已审核通过记录中的缺勤总次数（不去重）。

---

## 三、统计口径

### 1. 数据统计对象

只统计：**已审核通过 (approved)** 的记录。

不纳入统计：
- 待审核 (pending)
- 审核未通过 (rejected)
- 未提交
- 已撤销 (cancelled)

### 2. 统计周期定义

| 周期 | 定义 | 示例（今天是4月10日） |
|------|------|------------------------|
| 近7天 | 昨天之前连续7天 | 4月3日 00:00 至 4月9日 23:59 |
| 近30天 | 昨天之前连续30天 | 3月11日 00:00 至 4月9日 23:59 |
| 总榜 | 系统启用至昨天 | 系统启用日至 4月9日 23:59 |

### 3. 趋势对比规则

| 周期 | 对比对象 |
|------|----------|
| 近7天 | 再往前的7天 |
| 近30天 | 再往前的30天 |
| 总榜 | 不显示趋势 |

### 4. 无记录班级处理

统计周期内无已审核通过记录的班级不上榜（不显示0）。

## 五、服务端设计

### 1. 数据库表设计

#### ranking_cache 表

```sql
CREATE TABLE ranking_cache (
    id INT AUTO_INCREMENT PRIMARY KEY,
    period_type VARCHAR(10) NOT NULL,       -- '7d', '30d', 'total'
    rank_type VARCHAR(10) NOT NULL,         -- 'score', 'rate', 'count'
    class_id INT NOT NULL,
    rank_position INT NOT NULL,             -- 排名（1开始）
    rank_value DECIMAL(10,4) NOT NULL,      -- 当前值（分数/比率/人次）
    trend_value DECIMAL(10,4),              -- 趋势值（近7天/30天）
    trend_rank VARCHAR(10),                 -- 趋势排名：'UP2', 'DOWN1', 'SAME', 'NEW'
    total_expected INT,                     -- 应到人次
    total_absent INT,                       -- 缺勤人次
    total_late INT,                         -- 迟到人次
    total_leave INT,                        -- 请假人次
    total_other INT,                        -- 其他人次
    calculated_at DATETIME NOT NULL,        -- 计算时间
    INDEX idx_period_rank (period_type, rank_type, rank_position)
);
```

#### ranking_summary 表（概览数据）

```sql
CREATE TABLE ranking_summary (
    id INT AUTO_INCREMENT PRIMARY KEY,
    period_type VARCHAR(10) NOT NULL,
    rank_type VARCHAR(10) NOT NULL,
    avg_value DECIMAL(10,4) NOT NULL,       -- 平均值
    avg_trend DECIMAL(10,4),                -- 平均值趋势
    top_class_id INT,                       -- 最高班级ID
    top_class_name VARCHAR(50),             -- 最高班级名称
    top_value DECIMAL(10,4),                -- 最高值
    top_trend DECIMAL(10,4),                -- 最高值趋势
    total_classes INT,                      -- 上榜班级数
    calculated_at DATETIME NOT NULL,
    UNIQUE KEY uk_period_rank (period_type, rank_type)
);
```

### 2. 定时任务

```python
# server/tasks/ranking_calculator.py

每天凌晨 02:00 执行

流程：
1. 获取所有已审核通过的记录
2. 计算每个班级在3个周期内的统计数据
3. 计算综合异常分数、缺勤率、缺勤人次
4. 计算趋势（与上一周期对比）
5. 排序并生成排名
6. 存入 ranking_cache 和 ranking_summary 表
```

## 八、注意事项

### 1. 同值排序规则

数值相同时：先按值排序 → 值相同按班级名称排序 → 保证稳定

### 2. 变化率边界处理

| 情况 | 处理 |
|------|------|
| 上期无数据 | 趋势值 `—`，趋势排名 `NEW` |
| 上期为0本期有值 | 趋势值 `—`，趋势排名 `NEW` |
| 本期上期都无法比较 | 趋势值 `—`，趋势排名 `—` |

### 3. 总榜特殊处理

- 不显示趋势值
- 不显示趋势排名
- 概览区不显示变化率
