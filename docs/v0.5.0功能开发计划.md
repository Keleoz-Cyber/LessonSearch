## 一、版本目标

本次版本主要实现以下内容：
1. 记名名单提交功能
2. 周名单汇总功能
3. 用户角色分级制度

本版本的核心目标是建立完整的周考勤处理流程，即：
**成员提交记名名单 → 管理员审核 → 系统汇总 → 导出周考勤表**

**重要说明：**
- ✅ 记名（name_check）任务需要提交审核
- ❌ 点名（roll_call）任务不需要提交
- 记名任务包含多个班级的完整考勤情况，适合提交汇总
- 点名任务是单人点名，主要用于快速点名场景，不参与周汇总

## 二、角色系统实现

### 2.1 角色分类

- **member（成员）**：普通用户，查看自己的数据，提交记录
- **admin（管理员）**：审核、导出、查看所有数据

**说明：**
- 程序员/运维无需账号，直接在服务端通过SQL/脚本进行系统维护
- 管理员身份通常是学习部部长等日常管理人员

### 2.2 实现细节

- 管理员账号：使用已注册账号设置 role='admin'
- 邀请码功能：仅用于注册认证，不区分角色
- User 表修改：增加 role 字段

### 2.3 权限控制

- API 层：中间件/依赖注入检查角色权限
- 前端层：根据角色显示/隐藏功能

### 2.4 详细权限矩阵

## 三、周次配置实现

### 3.1 规则

- 权限：程序员/运维在服务端设定第1周起始日期
- 学期结束处理：手动重置（创建新学期配置）
- 周次计算：根据起始日期动态计算当前周次

### 3.2 周次计算逻辑

**规则：每周一为新的一周开始**

```python
# Python 计算示例
from datetime import datetime, timedelta

def calculate_week_number(current_date: datetime, start_date: datetime) -> int:
    """
    计算当前日期是第几周
    
    Args:
        current_date: 当前日期
        start_date: 第1周起始日期（必须是周一）
    
    Returns:
        周次（整数，从1开始）
    """
    # 计算相差天数
    days_diff = (current_date - start_date).days
    
    # 如果在起始日期之前，返回0
    if days_diff < 0:
        return 0
    
    # 计算周次（每周7天）
    week_number = (days_diff // 7) + 1
    
    return week_number

# 示例
start_date = datetime(2026, 3, 2)  # 2026年3月2日（周一）
print(calculate_week_number(datetime(2026, 3, 2), start_date))  # 第1周
print(calculate_week_number(datetime(2026, 3, 8), start_date))  # 第1周（周日）
print(calculate_week_number(datetime(2026, 3, 9), start_date))  # 第2周（下周一）
```

**注意事项：**
- `start_date` 必须是周一
- 周一到周日为同一周
- 跨年自然处理（无需特殊逻辑）

### 3.3 新学期设置流程（服务端操作）

程序员/运维通过SSH登录服务器，执行SQL：

```bash
# 1. 登录MySQL
mysql -u root -p lesson_search

# 2. 禁用旧学期配置

# 3. 创建新学期配置（注意：start_date 必须是周一）

    start_date,
    semester_name,
    is_active
) VALUES (
    '2026-09-07',           -- 2026年9月7日（周一）
    '2026-2027学年第一学期', -- 学期名称

# 4. 验证

```

**重要：**
- `start_date` 必须选择周一
- 系统会根据这个日期计算所有周次

操作时机：
- 学期开始前（开学第一周之前）
- 系统会自动使用最新的 is_active=TRUE 配置计算周次

### 3.4 数据库设计

```sql
CREATE TABLE week_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    start_date DATE NOT NULL,
    semester_name VARCHAR(50),
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**说明：**
- is_active: 标识当前活跃学期配置（只有一个为 TRUE）
- start_date: 第1周起始日期（必须是周一）
- 新学期设置：创建新记录并设置 is_active=TRUE，旧记录自动设为 FALSE
- 周次计算：使用当前 is_active=TRUE 的配置计算
- 历史记录：保留历史学期配置用于查询历史数据

### 4.5 本地数据隔离与多用户登录

**问题场景：**

**规则1：查询时过滤用户ID**

**规则2：切换账号时数据隔离**

```
用户A登录：
- 本地数据库包含 user_id=A 的任务、记录、提交
- App 只显示 user_id=A 的数据

切换到用户B登录：
- 本地数据库仍包含 user_id=A 的数据（不清空）
- App 只显示 user_id=B 的数据
- user_id=A 的数据仍在，但不可见
```

**规则3：未登录数据处理**

```
未登录时：
- 创建任务时 user_id = NULL
- 本地数据正常使用

登录后：
- 将所有 user_id=NULL 的数据绑定到当前账号
- user_id=NULL → user_id=当前用户ID
```

**绑定逻辑：**

```dart
    ..where((t) => t.userId.isNull()))
    .write(TasksCompanion(
      userId: Value(userId),
    ));
  
  // 绑定提交
  await (update(submissions)
    ..where((s) => s.userId.isNull()))

}
```

```python
async def create_submission(
    body: CreateSubmissionRequest,
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db)
):
    # 1. 验证任务归属
    for task_id in body.task_ids:
        task = db.query(AttendanceTask).filter(
            AttendanceTask.id == task_id
        ).first()
        
        if not task:
            raise HTTPException(status_code=404, detail=f"任务 {task_id} 不存在")
        
        # 关键：验证任务是否属于当前用户
        if task.user_id != current_user.id:
            raise HTTPException(
                status_code=403, 
                detail=f"任务 {task_id} 不属于当前用户，无权提交"
            )
    
    # 2. 获取所有记录
    records = db.query(AttendanceRecord).filter(
        AttendanceRecord.task_id.in_(body.task_ids)
    ).all()
    
    # 3. 验证记录归属（通过任务间接验证）
    for record in records:
        task = db.query(AttendanceTask).filter(
            AttendanceTask.id == record.task_id
        ).first()
        
        if task.user_id != current_user.id:
            raise HTTPException(
                status_code=403,
                detail="包含不属于自己的考勤记录，无法提交"
            )
    
    # 4. 创建提交
    submission = Submission(
        user_id=current_user.id,
        week_number=body.week_number,
    )
    db.add(submission)
    db.flush()
    
    # 5. 关联记录
    for record in records:
        sr = SubmissionRecord(
            submission_id=submission.id,
            record_id=record.id,
        )
        db.add(sr)
```

### 5.2 撤销记录

- 用户可重新提交新的记录

## 六、Excel导出模板格式

### 6.1 文件名

**文件名：** 第X周周考勤表.xlsx

### 6.2 列结构（7列）

| 列号 | 列名 | 数据类型 | 说明 |
|------|------|----------|------|
| 1 | 序号 | 整数 | 自动编号（从1开始） |
| 2 | 姓名 | 字符串 | 学生姓名 |
| 3 | 班级 | 字符串 | 班级名称（如：电信2402） |
| 4 | 学号 | 字符串 | 完整学号（如：241040200219） |
| 5 | 迟到/早退 | 整数 | 迟到或早退次数 |
| 6 | 旷课 | 整数 | 旷课次数 |

### 6.3 字段映射关系

| Excel列名 | APP记录状态 | 说明 |
|-----------|-------------|------|
| 迟到/早退 | late | APP中标记为"迟到"状态的记录数 |
| 旷课 | absent | APP中标记为"缺勤"状态的记录数 |
| 请假 | - | 不计入Excel导出 |
| 其他 | - | 不计入Excel导出 |
| 已到 | - | 不计入Excel导出 |

### 6.4 数据聚合规则

**重要：按学生聚合统计**

同一个学生一周内可能被多次记名（不同班级/不同任务），Excel 导出时：

**聚合规则：**
- 每个学生只显示一行
- 统计该学生本周所有审核通过的迟到、旷课次数
- 按学生ID（student_id）分组

**示例：**

```
张三本周被记名3次：
- 周一：计科2401班，迟到
- 周三：电信2402班，旷课
- 周五：物联网2401班，迟到

Excel 导出结果：
姓名：张三
班级：计科2401（取第一次记名的班级）
学号：241040200123
迟到/早退：2
旷课：1
累计：2
```

**SQL 聚合示例：**
```sql
SELECT 
    s.id as student_id,
    s.name,
    s.student_no,
    c.display_name as class_name,
    COUNT(CASE WHEN r.status = 'late' THEN 1 END) as late_count,
    COUNT(CASE WHEN r.status = 'absent' THEN 1 END) as absent_count
FROM students s
JOIN attendance_records r ON s.id = r.student_id
JOIN submissions sub ON r.id IN (
    SELECT sr.record_id 
    FROM submission_records sr 
    WHERE sr.submission_id IN (
        SELECT id FROM submissions 
        WHERE week_number = 3 AND status = 'approved'
    )
)
JOIN classes c ON s.class_id = c.id
WHERE r.status IN ('late', 'absent')
GROUP BY s.id  -- 按学生ID分组
ORDER BY 
    m.short_name ASC,
    CAST(c.class_code AS SIGNED) ASC,
    s.student_no ASC;
```

### 6.5 累计公式

```
=ROUNDDOWN(迟到/早退/2 + 旷课, 0)
```

即：
- 2次迟到 = 1次累计
- 1次旷课 = 1次累计

### 6.5 数据排序规则

**1. 专业排序**（按 major.short_name 字母顺序）：
- 空信（Kongxin）
- 通信（Tongxin）
- 物联网（Wulianwang）
- 电信（XinDian）
- 计科（JiKe）
- 实际排序：使用 ORDER BY major.short_name ASC

**2. 班级排序**（按 class.class_code 数值升序）：
- 电信2401 → 电信2402 → 电信2403
- 实际排序：使用 ORDER BY CAST(class.class_code AS SIGNED) ASC

**3. 学号排序**（按 student.student_no 升序）：
- 实际排序：使用 ORDER BY student.student_no ASC

**完整 SQL 排序示例：**
```sql
SELECT 
    s.name, 
    c.display_name as class_name, 
    s.student_no,
    ...
FROM students s
JOIN classes c ON s.class_id = c.id
JOIN majors m ON c.major_id = m.id
ORDER BY 
    m.short_name ASC,          -- 专业排序
    CAST(c.class_code AS SIGNED) ASC,  -- 班级排序
    s.student_no ASC           -- 学号排序
```

### 7.2 数据库迁移步骤

**1. User 表添加 role 和 real_name 字段**
```sql
-- 为现有用户添加角色字段，默认为 member
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'member';

```

**说明：**
- role: 用户角色（'member' 或 'admin'）
- real_name: 真实姓名（登录后必填）
- 现有用户的 real_name 为 NULL，登录后需要填写

**2. 设置管理员账号**
```sql

-- 方式二：通过用户ID设置

```

**说明：**
- role 字段只有两个值：'member'（默认）和 'admin'
- 程序员/运维无需账号，直接通过SSH操作服务器

**3. 创建新表（不影响现有数据）**
```sql
-- 创建 week_config 表
CREATE TABLE week_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    start_date DATE NOT NULL,
    semester_name VARCHAR(50),
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 创建 submissions 表
CREATE TABLE submissions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    week_number INT NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    reviewer_id INT,
    review_time DATETIME,
    review_note TEXT,
    submitted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (reviewer_id) REFERENCES users(id),
    INDEX idx_user_week (user_id, week_number),
    INDEX idx_status (status),
    INDEX idx_week (week_number)
);

-- 创建 submission_records 关联表
CREATE TABLE submission_records (
    id INT PRIMARY KEY AUTO_INCREMENT,
    submission_id INT NOT NULL,
    record_id INT NOT NULL,
    FOREIGN KEY (submission_id) REFERENCES submissions(id),
    FOREIGN KEY (record_id) REFERENCES attendance_records(id)
);

-- 创建 week_exports 表（记录周次导出状态）
CREATE TABLE week_exports (
    id INT PRIMARY KEY AUTO_INCREMENT,
    week_number INT NOT NULL,
    exported_by INT NOT NULL,
    exported_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (exported_by) REFERENCES users(id),
    UNIQUE KEY uk_week (week_number)
);
```

## 十、实时公告功能设计

### 10.1 数据库设计

```sql
CREATE TABLE announcements (
    id INT PRIMARY KEY AUTO_INCREMENT,
    version INT NOT NULL,
    title VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    target_role VARCHAR(20) DEFAULT 'all',  -- all/member/admin
    is_active BOOLEAN DEFAULT TRUE,
    created_by INT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES users(id),
    INDEX idx_version (version),
    INDEX idx_active (is_active)
);
```

**字段说明：**
- version: 公告版本号（客户端用此判断是否需要显示）
- target_role: 目标角色（all=所有人，member=仅成员，admin=仅管理员）
- is_active: 是否激活（false则不显示）

### 10.2 API 设计

**1. 获取最新公告**

```python
@router.get("/announcement")
async def get_announcement(
    db: Session = Depends(get_db)
):
    """客户端启动时调用，获取最新公告"""
    announcement = db.query(Announcement).filter(
        Announcement.is_active == True
    ).order_by(Announcement.version.desc()).first()
    
    if not announcement:
        return {"version": 0, "title": "", "content": ""}
    
    return {
        "version": announcement.version,
        "title": announcement.title,
        "content": announcement.content,
        "updated_at": announcement.updated_at.isoformat()
    }
```

**2. 获取定向公告（带角色筛选）**

```python
@router.get("/announcement/for-user")
async def get_announcement_for_user(
    current_user: User = Depends(get_current_user_optional),
    db: Session = Depends(get_db)
):
    """获取针对当前用户角色的公告"""
    user_role = current_user.role if current_user else None
    
    announcement = db.query(Announcement).filter(
        Announcement.is_active == True,
        (Announcement.target_role == 'all') | 
        (Announcement.target_role == user_role)
    ).order_by(Announcement.version.desc()).first()
    
    return {...}
```

**注意：** 公告管理由程序员/运维在服务端通过SQL/脚本操作，不提供客户端管理API。

### 10.3 公告管理方式（服务端操作）

**方式二：Python 脚本（推荐）**

创建 `scripts/manage_announcement.py`：

```python
import sys
sys.path.append('/opt/lesson-search/server')
from database import SessionLocal
from models import Announcement

def create_announcement(version: int, title: str, content: str, target_role: str = 'all'):
    db = SessionLocal()
    try:
        announcement = Announcement(
            version=version,
            title=title,
            content=content,
            target_role=target_role,
            created_by=1  # 使用任意管理员ID，如第一个管理员的ID
        )
        db.add(announcement)
        db.commit()
        print(f"✅ 公告创建成功: v{version} - {title}")
    finally:
        db.close()

if __name__ == '__main__':
    if len(sys.argv) < 4:
        print("用法: python manage_announcement.py <version> <title> <content>")
        sys.exit(1)
    
    create_announcement(
        version=int(sys.argv[1]),
        title=sys.argv[2],
        content=sys.argv[3]
    )
```

### 10.3 客户端实现

**1. 移除硬编码，改为从服务端获取**

```dart
// lib/core/announcement/announcement_service.dart

class AnnouncementService {
  static const _prefKey = 'dismissed_announcement_version';
  static const _cacheKey = 'cached_announcement';
  
  /// 检查是否需要显示公告
  static Future<void> checkAndShow(BuildContext context) async {
    try {
      // 从服务端获取最新公告
      final response = await http.get(
        Uri.parse('https://api.keleoz.cn/api/announcement')
      );
      
      if (response.statusCode != 200) {
        // 网络失败，使用缓存
        _showCached(context);
        return;
      }
      
      final data = json.decode(response.body);
      final version = data['version'] as int;
      
      // 缓存公告
      final prefs = await SharedPreferences.getInstance();
      await prefs.setString(_cacheKey, json.encode(data));
      
      // 检查是否已看过
      final dismissed = prefs.getInt(_prefKey) ?? 0;
      if (dismissed >= version) return;
      
      // 显示公告
      _showDialog(context, data);
      
    } catch (e) {
      // 出错时使用缓存
      _showCached(context);
    }
  }
  
  static Future<void> _showCached(BuildContext context) async {
    final prefs = await SharedPreferences.getInstance();
    final cached = prefs.getString(_cacheKey);
    if (cached == null) return;
    
    final data = json.decode(cached);
    final dismissed = prefs.getInt(_prefKey) ?? 0;
    if (dismissed >= data['version']) return;
    
    _showDialog(context, data);
  }
  
  static Future<void> _showDialog(BuildContext context, Map<String, dynamic> data) async {
    await showDialog(
      context: context,
      builder: (ctx) => AlertDialog(
        title: Text(data['title']),
        content: SingleChildScrollView(
          child: Text(data['content']),
        ),
        actions: [
          TextButton(
            onPressed: () => Navigator.pop(ctx),
            child: const Text('关闭'),
          ),
        ],
      ),
    );
    
    // 记录已看过
    final prefs = await SharedPreferences.getInstance();
    await prefs.setInt(_prefKey, data['version']);
  }
}
```

### 10.5 更新流程对比

**旧流程（硬编码）：**
```
1. 修改 announcement_config.dart
2. announcementVersion++
3. 编译打包 App
4. 发布新版本
5. 用户下载更新
6. 用户打开 App 看到新公告
```

**新流程（实时公告）：**
```
1. 数据库插入新公告（version=19）
2. 用户打开 App
3. App 获取最新公告
4. 显示新公告
```
